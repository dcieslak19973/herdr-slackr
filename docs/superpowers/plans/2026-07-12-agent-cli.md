### Task 1: `src/cli.rs` — mentions/feed + config fallback + skill-path/skill-install

**Files:**
- Create: `src/cli.rs`, `skills/herdr-slackr/SKILL.md`
- Modify: `src/main.rs` (dispatch `mentions|feed|skill-path|skill-install` → `cli::run(args) -> ExitCode` before the HERDR guard/pane launch; `--version` unchanged), `src/lib.rs` (`pub mod cli;`)
- Test: `tests/agent_cli.rs`

**Interfaces:**
- Consumes: `rest::{Rest, auth_self, list_conversations, history, users, RestError}`, `entities::{resolve, is_mention}`, `model::{Message, Conversation, ConvKind, ts_cmp}`, `config::{plugin_config_in, PluginConfig}`, `tokens::resolve`, `app`'s `ts_to_hhmm` if reusable (check visibility; else a local copy with a doc note).
- Produces: `cli::run(args: Vec<String>) -> std::process::ExitCode`.

**Behavior (spec §CLI, encode exactly):**
- Config dir: `HERDR_PLUGIN_CONFIG_DIR` env, else `~/.config/herdr/plugins/config/dcieslak19973.slackr` (`${HOME:-USERPROFILE}`); neither yields a readable config → exit 1 naming BOTH paths tried. Tokens resolved as the pane does (env → tokens.toml in that dir).
- `mentions [--json] [--limit n]` (default 20): auth_self → list_conversations filtered to the config's channels (+ IMs/MPIMs when `dms`) → history(50) per conv → filter `is_mention(msg, kind, self_id, keywords)` → sort newest-first by `ts_cmp` → cap. Human row `#chan  @author  HH:MM  text` with entities resolved via the users/conv caches; `--json` = array of `{conversation, conv_id, author, author_id, ts, text}` (resolved text; raw available as `text_raw`).
- `feed [--channel "#name"] [--json] [--limit n]` (default 20): same scan, no mention filter; `--channel` must name a configured channel (else exit 1 naming it and listing the configured set).
- Pure/testable split: `fn select_mentions(msgs, kinds, self_id, keywords) -> Vec<..>` / `fn format_row(..) -> String` / `fn config_dir(env_fn, home_fn) -> Result<PathBuf, String>` are pure and unit-tested; the REST orchestration is the thin edge.
- `skill-path` + `skill-install [--target|--project] [--copy] [--force]`: port reviewr's `resolve_skill_source`/`skill_install` (read them; adapt names — skill dir `skills/herdr-slackr`, default target `~/.claude/skills/herdr-slackr`, `--project` → `.agents/skills/herdr-slackr`; hint block prints a slackr-appropriate CLAUDE.md line: "Slack triage happens via herdr-slackr — when the user asks about mentions or a channel, run `herdr-slackr mentions --json` / `herdr-slackr feed --channel …`").
- SKILL.md: write per spec §Skill (frontmatter name `herdr-slackr`, description triggering on mention-checking/channel-context/slackr-setup; body: triage/context commands with --json guidance, READ-ONLY rule + MCP deferral, binary discovery order, setup/troubleshoot incl. log path and degraded states, pane-control invocations).
- Rate limits: `RestError::RateLimited(s)` → exit 1, one stderr line "slackr: slack rate limit — retry in {s}s". Other RestErrors → exit 1 with the Slack error name.
